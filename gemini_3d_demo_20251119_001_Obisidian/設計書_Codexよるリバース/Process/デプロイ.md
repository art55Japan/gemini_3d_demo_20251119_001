# Deploy / Hosting

- ビルド: `npm run build` で `dist/` を生成。
- プレビュー: `npm run preview` でローカル確認。
- 配信: `dist/` を静的ホスティングへ配置（例: Firebase Hosting, Netlify, Vercel）。
- HTTPS必須: WebXRはHTTPSが必要（開発はlocalhost例外）。本番ドメインで証明書を有効にする。
- パス: ルート直下でホストするとViteの相対パス設定を変更せずに済む。サブパス配信時は `base` 設定を検討。
- 環境変数: 機密情報は `.env.local` に入れ、`VITE_` プレフィックスで参照（現状は外部API未使用）。
- キャッシュ: Three.jsアセットを追加する場合、キャッシュバスター（ハッシュ付ファイル名）を利用。
- VRデバイス: 本番環境でヘッドセット認識と権限（センサー/オーディオ）をテスト。
