# Conventions

## コーディング
- ESモジュール前提。クラス名はPascalCase、インスタンスはcamelCase、定数はSCREAMING_SNAKE_CASE。
- 4スペースインデント、セミコロン、シングルクォート。外部→ローカルの順でimportを並べる。
- 条件分岐を減らし、小さなクラス/コンポーネントに分解（`Entity` 継承パターン、`Player` の物理/戦闘/衝突コンポーネント分割）。
- Three.js設定は `Game`、UIは `SaveLoadUI` に寄せ、責務を混在させない。

## コミット/PR
- コミットメッセージは命令形・短め。スコープ指定や日本語も可（例: `Refactor: simplify PlayerCombat`）。
- PRでは: 目的と変更概要、実行したコマンド例（`npx vitest run`, `npm run build`）、UI変更時はVR/デスクトップのスクショ or 動画、関連Issueがあれば紐付け。
- コントロール/セーブ形式/ビルド手順を変えたらドキュメントを更新。
