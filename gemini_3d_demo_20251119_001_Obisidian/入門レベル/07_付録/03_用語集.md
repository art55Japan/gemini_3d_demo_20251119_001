# 用語集

> [!abstract] 概要
> 本ドキュメントで使用する用語をアルファベット順・五十音順にまとめています。

---

## A-Z

### AABB (Axis-Aligned Bounding Box)
軸に沿った境界ボックス。衝突判定に使用される最もシンプルな方法。回転しない矩形で物体を囲み、重なりを検出する。
→ 関連: [[05_ゲームメカニクス編/01_物理演算_PlayerPhysics.js#AABB衝突判定]]

### API (Application Programming Interface)
ソフトウェア同士が情報をやり取りするための取り決め。例えば Web Audio API は、ブラウザで音を鳴らすための API。

### async/await
JavaScript の非同期処理を同期的に書ける構文。`async` 関数内で `await` を使うと、Promise の完了を待てる。

### Canvas
HTML要素の一つ。JavaScript でグラフィックスを描画するための領域。Three.js は内部で Canvas を使用する。

### class（クラス）
オブジェクトの設計図。プロパティ（データ）とメソッド（処理）をまとめたもの。
→ 関連: [[03_設計パターン編/01_エンティティの基礎_Entity.js]]

### constructor（コンストラクタ）
クラスからインスタンスを作成する際に呼ばれる特別なメソッド。初期化処理を行う。

### delta（デルタ）
前フレームからの経過時間（秒）。フレームレートに依存しない動きを実現するために使用。

### DOM (Document Object Model)
HTML文書をプログラムから操作するための仕組み。`document.getElementById()` などで要素を取得できる。

### ES Modules
JavaScript の公式モジュールシステム。`import`/`export` でファイル間の依存関係を管理する。
→ 関連: [[01_基礎編/03_エントリーポイント_main.js]]

### extends
クラスの継承を行うキーワード。親クラスの機能を引き継いで新しいクラスを作成する。

### fps (Frames Per Second)
1秒間に描画されるフレーム数。ゲームでは60fpsが標準的。

### Geometry（ジオメトリ）
Three.js における3Dオブジェクトの形状データ。頂点の位置や面の情報を持つ。
→ 関連: [[02_Three.js概念図解#Geometry形状]]

### getter/setter
プロパティへのアクセスをカスタマイズするための特別なメソッド。
→ 関連: [[03_設計パターン編/01_エンティティの基礎_Entity.js#getter--setter]]

### Group（グループ）
Three.js で複数のオブジェクトをまとめるコンテナ。親子関係を作れる。

### JSON (JavaScript Object Notation)
データ交換用のテキストフォーマット。JavaScriptのオブジェクトに似た構造。LocalStorageに保存する際に使用。
→ 関連: [[06_システム編/02_セーブ機能_SaveManager.js]]

### LocalStorage
ブラウザにデータを永続保存するAPI。約5MBまで保存可能で、文字列のみ格納できる。

### Material（マテリアル）
Three.js における3Dオブジェクトの見た目（色、質感、透明度など）。

### Mesh（メッシュ）
Three.js で描画される3Dオブジェクト。Geometry（形状）とMaterial（材質）を組み合わせて作成。

### npm (Node Package Manager)
JavaScript のパッケージ管理ツール。外部ライブラリのインストールや管理を行う。
→ 関連: [[01_基礎編/02_パッケージ管理_package.json]]

### OOP (Object-Oriented Programming)
オブジェクト指向プログラミング。データと処理をオブジェクトとしてまとめる設計手法。

### Oscillator（オシレーター）
Web Audio API で音波を生成するノード。sine、square、triangle、sawtooth などの波形を選べる。
→ 関連: [[06_システム編/03_サウンド_AudioManager.js]]

### Promise
非同期処理の結果を表すオブジェクト。成功（resolve）または失敗（reject）の状態を持つ。

### Raycaster（レイキャスター）
Three.js でマウス位置から3D空間への光線を飛ばし、交差判定を行う機能。
→ 関連: [[06_システム編/01_ビルドシステム_BuildSystem.js]]

### Renderer（レンダラー）
Three.js で3Dシーンを2D画面に描画するコンポーネント。WebGLRenderer が一般的。

### Scene（シーン）
Three.js で全てのオブジェクト（メッシュ、ライト、カメラなど）を配置するコンテナ。

### super
親クラスのコンストラクタやメソッドを呼び出すキーワード。

### this
現在のオブジェクト（インスタンス）を指すキーワード。クラス内でプロパティやメソッドにアクセスする際に使用。

### Three.js
WebGL を簡単に扱うための JavaScript ライブラリ。3Dグラフィックスをブラウザで描画できる。

### Vector3
Three.js の3次元ベクトルクラス。位置、方向、速度などを表現する。

### Vite（ヴィート）
高速な開発サーバーとビルドツール。ES Modules をネイティブサポートし、ホットリロードが速い。

### WebGL
ブラウザでGPUを使った3Dグラフィックスを描画するためのAPI。Three.js の内部で使用される。

## あ行

### アニメーションループ
ゲームのメインループ。毎フレーム呼び出され、更新処理と描画を行う。`requestAnimationFrame` で実装する。
→ 関連: [[02_Three.js入門編/01_ゲームの心臓部_Game.js]]

### イベント駆動
イベント（クリック、キー入力など）が発生したときに処理を実行する設計パターン。
→ 関連: [[02_Three.js入門編/02_入力システム_Input.js]]

### インスタンス
クラスから生成された具体的なオブジェクト。`new` キーワードで作成する。

### エンティティ
ゲーム内のオブジェクト（プレイヤー、敵、ブロックなど）を表す基本単位。
→ 関連: [[03_設計パターン編/01_エンティティの基礎_Entity.js]]

### オーバーライド
親クラスのメソッドを子クラスで再定義すること。

---

## か行

### 関数型プログラミング
データの変換を関数の連鎖で表現する設計手法。`map`、`filter`、`reduce` などを多用する。

### 継承
既存のクラス（親クラス）の機能を引き継いで新しいクラス（子クラス）を作成すること。

### コールバック
他の関数に引数として渡される関数。イベント発生時や非同期処理完了時に呼び出される。

### コンポーネントベース
機能を独立したコンポーネント（部品）に分割し、組み合わせてオブジェクトを構成する設計。
→ 関連: [[03_設計パターン編/03_プレイヤーの設計_Player.js]]

---

## さ行

### シーングラフ
3Dオブジェクトの親子関係を木構造で表現したもの。親の変換（移動・回転）が子に伝播する。

### ステートパターン
オブジェクトの状態を別クラスとして表現し、状態ごとの振る舞いをカプセル化するパターン。
→ 関連: [[05_ゲームメカニクス編/02_戦闘システム_PlayerCombat.js]]

### スプライト
2Dの画像を3D空間に配置したもの。常にカメラの方を向く。

---

## た行

### データ駆動
設定値やルールをデータ（オブジェクトや配列）として定義し、コードの変更なしに動作を変更できる設計。

### テクスチャ
3Dオブジェクトの表面に貼る画像。Material の `map` プロパティで設定する。

### 透視投影
遠くのものが小さく見える一般的な3D投影方法。PerspectiveCamera で実現。

---

## な行

### 法線（ノーマル）
面に垂直な方向を表すベクトル。光の反射計算や衝突判定で使用。

### ノックバック
攻撃を受けた際に後退する演出。物理演算で速度を加算して実現。

---

## は行

### パラダイム
プログラミングの考え方や設計手法。OOP、関数型、イベント駆動などがある。
→ 関連: [[05_パラダイム早見表]]

### ポリモーフィズム（多態性）
同じインターフェースで異なる実装を呼び出せる性質。例えば `entity.update()` が Entity の種類によって異なる動作をする。

## ま行

### マテリアル
→ Material を参照

### メソッドチェーン
メソッドの戻り値を使って連続的にメソッドを呼び出す書き方。`array.filter().map().forEach()` など。

## や行

### 要素
配列の中の個々のデータ。`array[0]` で最初の要素にアクセスできる。

---

## ら行

### ラジアン
角度の単位。360度 = 2π ラジアン。Three.js では角度をラジアンで指定する。
- 変換: `degrees * Math.PI / 180`

### ライフサイクル
オブジェクトの生成から破棄までの一連の流れ。`enter()` → `update()` → `exit()` など。

### レイキャスト
光線を飛ばしてオブジェクトとの交差を検出する技術。マウスピッキングに使用。
→ Raycaster を参照

### レンダリング
3Dシーンを2D画像として画面に描画すること。
